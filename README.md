#  Online Book Store Database Project

##  Project Overview
The **Online Book Store Database Project** is a relational database system designed using SQL to simulate the backend of an online bookstore.  
It manages books, customers, orders, payments, and inventory while maintaining data integrity and realistic relationships between tables.

This project is ideal for **SQL practice, academic projects, and portfolio demonstration**.

---

##  Technologies Used
- SQL
- MySQL / PostgreSQL (any relational DBMS)
- Database normalization concepts

---

##  Database Features
- Structured and normalized database design
- Use of **Primary Keys** and **Foreign Keys**
- Data integrity using **constraints**
- Realistic relationships between entities
- Supports real-world bookstore operations

---

##  Database Tables
- **Books** – stores book details (title, author, price, stock)
- **Customers** – stores customer information
- **Orders** – tracks customer purchases
- **Order_Details** – manages multiple books per order
- **Payments** – stores payment records
- **Categories** – classifies books by genre/type

---

## Relationships
- One customer can place multiple orders
- One order can contain multiple books
- Each book belongs to a category
- Orders are linked with payment details

---

##  Key SQL Concepts Used
- `CREATE`, `INSERT`, `UPDATE`, `DELETE`
- `PRIMARY KEY`, `FOREIGN KEY`
- `UNIQUE`, `NOT NULL`, `CHECK`
- `JOIN` (INNER JOIN)
- `GROUP BY`, `ORDER BY`
- Aggregate functions (`COUNT`, `SUM`, `AVG`)
- Subqueries
